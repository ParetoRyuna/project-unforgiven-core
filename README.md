Unlike standard auctions that treat all addresses equally (making them vulnerable to Sybil/Bot attacks), Project UNFORGIVEN introduces a **Sybil-Resistant Pricing Layer**.

### 1. Hybrid ZK-Identity Architecture (PayFi Ready)
We utilize a sophisticated **Off-chain Attestation + On-chain Verification** model to protect user privacy while ensuring economic integrity:

* **Privacy-First Verification:** User behavioral data (e.g., Spotify/Apple Music footprints) remains off-chain.
* **ZK-Logic Placeholder:** The protocol is designed to verify **Cryptographic Attestations** (via Ed25519 signature validation) generated by trusted execution environments or ZK-Proof generators.
* **Dynamic $\alpha$ Scoring:** Each participant is assigned an Identity Score ($\alpha$). High $\alpha$ (True Fans) receive a **Loyalty Discount**, while Low $\alpha$ (Anonymous Bots) trigger an exponential price penalty.

### 2. J-Curve Price Protection
The protocol calculates the **Total Access Cost** ($C_{access}$) in real-time on Solana:

$$C_{access} = P_0 + D(\alpha, t, k)$$

Where:
- $P_0$ is the base price.
- $D$ is the dynamic adjustment factor.
- If demand exceeds the target velocity ($k > rate \times t$), the **J-Curve** mechanism triggers an exponential price spike, making botting economically suicidal.
